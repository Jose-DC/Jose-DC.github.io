Caso de práctica profesional

Plataforma Docker Lab

Laboratorio de infraestructura para ordenar el despliegue de servicios Docker, observar su operación y revisar imágenes antes de reutilizarlas en nuevas pruebas.

Abrir repositorio Volver a proyectos

Situación

Problema

Preparar servicios Docker manualmente repite tareas como organizar archivos, seleccionar variables, conectar redes y revisar el estado de cada componente.

Propuesta

Qué resuelve

Reúne plantillas reutilizables, módulos de operación y controles básicos para hacer el flujo de laboratorio más consistente y fácil de revisar.

Alcance real

Estado

Laboratorio funcional validado con Docker Compose. No se presenta como una plataforma productiva terminada ni como un servicio expuesto a Internet.

Recorrido de la práctica

Del stack a la evidencia operacional

Las capturas siguientes muestran el flujo de trabajo del laboratorio. Son evidencias de práctica y fueron seleccionadas sin credenciales, dominios privados ni accesos administrativos.

  1. 01

    Preparar

    Revisar las plantillas disponibles

    Organicé el módulo de automatización para consultar combinaciones de aplicación y base de datos con make stacks antes de generar un entorno de prueba.

  2. 02

    Operar

    Revisar stacks y contenedores

    Incorporé Portainer como vista operativa para revisar inventario, estado de contenedores, imágenes, volúmenes y redes durante las pruebas.

  3. 03

    Validar

    Confirmar la recolección de métricas

    Configuré Prometheus para comprobar que cAdvisor, Node Exporter y el servidor de métricas estuvieran en estado UP antes de analizar los dashboards.

  4. 04

    Observar

    Analizar el comportamiento del host

    Centralicé en Grafana métricas de RAM, CPU, disco, red y carga para observar el comportamiento del servidor durante las pruebas.

  5. 05

    Revisar

    Escanear imágenes con Trivy

    Agregué un módulo con Trivy para revisar vulnerabilidades y conservar reportes locales rotados antes de reutilizar una imagen en otra prueba.

Aprendizajes

Lo que me dejó este laboratorio

  • Separar la generación de entornos de su operación hace más fácil repetir y revisar las pruebas.
  • Validar los targets antes de mirar dashboards evita interpretar métricas incompletas.
  • El monitoreo y el análisis de imágenes aportan evidencia útil para tomar decisiones técnicas.
  • Documentar variables y módulos desde el inicio reduce pasos manuales al reutilizar el laboratorio.
