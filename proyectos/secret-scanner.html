Capítulo 01 - Automatización y seguridad básica

Secret scanner para repositorios

Herramienta CLI en Python para revisar una carpeta antes de subirla a GitHub y detectar posibles secretos, archivos sensibles o patrones riesgosos. Es un proyecto pequeño, útil y alineado con buenas prácticas DevOps.

Problema

Cuando se trabaja con repositorios personales o laboratorios, es fácil subir por error archivos `.env`, tokens, claves, credenciales de prueba o configuraciones sensibles.

Objetivo

Crear una herramienta simple que escanee archivos, ignore carpetas innecesarias y entregue un reporte claro antes de hacer commit o push.

Stack

  • Python para la CLI y reglas de detección.
  • Git para integrarlo al flujo de trabajo.
  • Markdown para reportes y documentación.

Evidencia esperada

  • README con instalación y uso.
  • Capturas del scanner encontrando archivos de ejemplo.
  • Casos de prueba con falsos positivos documentados.

Próximos pasos

  1. Definir reglas iniciales: `.env`, tokens, claves privadas y URLs sensibles.
  2. Crear salida en consola con severidad y ruta del archivo.
  3. Agregar modo `--report` para generar Markdown.

Estado

En construcción. Este capítulo se irá actualizando con comandos, capturas y enlace al repositorio cuando el proyecto esté listo.
