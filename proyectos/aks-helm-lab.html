Capítulo 02 - Cloud y Kubernetes

AKS + Helm lab

Laboratorio para documentar el flujo de despliegue de una aplicación demo en Kubernetes usando contenedores, Helm y conceptos básicos de operación en AKS.

Problema

Muchos portafolios dicen “Kubernetes” sin mostrar el proceso real: manifiestos, valores de Helm, comandos, logs, errores encontrados y decisiones de despliegue.

Objetivo

Construir un laboratorio claro que muestre cómo se empaqueta, despliega, revisa y documenta una app demo en un entorno Kubernetes.

Stack

  • Azure Kubernetes Service como contexto cloud.
  • Docker para contenerizar la aplicación.
  • Helm para parametrizar el despliegue.
  • Kubectl para diagnóstico y revisión de estado.

Evidencia esperada

  • Arquitectura simple del laboratorio.
  • Comandos principales de despliegue y rollback.
  • Capturas o salidas de `kubectl get pods`, logs y eventos.

Próximos pasos

  1. Definir una app demo pequeña y fácil de ejecutar.
  2. Crear chart Helm con `values.yaml` documentado.
  3. Agregar sección de troubleshooting real.

Estado

Planeado. Este capítulo quedará como plantilla para documentar el laboratorio cuando empiece la implementación.
